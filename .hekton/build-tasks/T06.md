## Task Description

Implement `vg_core::traits::PolicyEngine` in `crates/vg-policy/src/` (currently an empty
stub crate). The trait: `fn load(layers: PolicyLayers) -> Result<Self, PolicyError> where
Self: Sized`, `fn classify_artefact(&self, hint: &ArtefactHint) -> HandlingClass`,
`fn classify_entity(&self, ty: EntityType) -> HandlingClass`,
`fn destination_allows_masked_only(&self, dest: &DestinationId) -> bool`,
`fn demask_allowed(&self, dest: Destination, actor: &Actor) -> bool`,
`fn version(&self) -> &str` (see `crates/vg-core/src/traits.rs` for the exact trait and
`api.rs`/`types.rs` for `PolicyLayers { global: PathBuf, repo: Option<PathBuf>, session:
Option<PathBuf> }`, `HandlingClass`, `Destination`, `DestinationId`, `Actor`).

Concretely:
1. Pick a config format for policy packs -- YAML (`serde_yaml`) or TOML (`toml`, already
   used elsewhere in this workspace) are both reasonable; add the dependency and write at
   least one example policy file (fixture) that `load` can actually parse, per the
   acceptance criterion.
2. 3-layer resolution: session overrides repo overrides global. Write a real test with
   all three layers present and overlapping keys, not just one layer.
3. Signed-pack verification is a deliberate **stub** in Phase 1 -- don't build real
   signature verification, just a clearly-marked placeholder that always accepts (comment
   why, and that Phase 2 replaces it).
4. **The one hard-deny rule that must be real, not stubbed:** `demask_allowed` must return
   `false` for `Destination::RemoteModelPrompt` and `Destination::ObservabilitySink`
   regardless of actor, in the default policy. Use
   `vg_core::conformance::assert_policy_engine_denies_hard_deny_destinations` -- this is
   the one piece of this task that's actually security-load-bearing, the rest is
   configuration plumbing.
