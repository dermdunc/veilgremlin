## Task Description

Wire the masking pipeline in `crates/vg-core/src/api.rs`: replace the `todo!()` bodies of
`scan()` and `mask()` with real implementations. Everything they compose already exists and is
merged: detectors (T03, `vg_detectors::all_detectors()`), keying (T04, `vg_core::keying`),
vault (T05, `vg_vault::Vault`), audit sink (T05b, `vg_audit::JsonlAuditSink`), policy engine
(T06, `vg_policy::LayeredPolicyEngine`), parsers (T08, `vg_parsers`). `vg-core` reaches them
only through its own trait objects (`Context { parsers, detectors }`, `Policy { engine, vault,
audit }`) -- do NOT add a vg-core dependency on any implementing crate except as
dev-dependencies for integration tests (the T04 precedent).

**A known contract-shape gap you will hit immediately, resolve it this way:** the frozen
`mask(input, policy, ns)` signature has no `Context` parameter, so `mask` cannot reach the
detectors/parsers that `scan` gets via `ctx`. Resolve via the contract-change protocol
(`docs/architecture/agent-factory-plan.md` section 6): add `ctx: &Context` to `mask` (making it
`mask(input, ctx, policy, ns)`), update `docs/architecture/interface-contracts.md` section 2
with a versioned note recording the change and why, and update the conformance stubs if they
reference the old shape. Do not smuggle detectors into `Policy` and do not have callers
pre-compute findings -- the explicit ctx parameter is the sanctioned fix.

Pipeline order inside `mask`:
1. Classify the artefact first: `policy.engine.classify_artefact(&input.hint)`. If `Block`
   (e.g. a `.env` file per the default policy fixture), write an `AuditEvent::Block` via
   `policy.audit`, and return a `MaskedPack` whose `text` is empty/refused with
   `stats.blocked_artefacts = 1` -- the artefact's content must never appear in the pack.
2. **Hard requirement (T06 review): artefact `Pass` means "send after entity masking", NEVER
   "skip detection".** Entity scanning below runs for every non-Block artefact regardless of
   its artefact class, or the `Pass` default becomes a fail-open leak.
3. Parse: pick the first parser whose `can_parse(&input.hint)` is true; run it for spans.
   **Hard requirement (T08 review): spans are enrichment only -- detectors scan the FULL raw
   buffer, not span-gated regions.** Two known under-spanning reproducers are documented in
   docs/decisions.md (a `#` inside a quoted YAML value; single-quoted YAML flow scalars); if
   you gate detection on spans those become missed-secret bugs. Pass spans through to
   `Detector::detect(buf, spans)` as the trait allows, but never restrict what gets scanned.
4. Detect: run every detector in `ctx.detectors` over the buffer; collect Findings.
5. Resolve overlapping findings before masking: when two findings' spans overlap, keep the
   more specific entity type over the generic one (e.g. an `Email` finding wins over an
   `entropy` `Secret` covering the same bytes; T03's docs anticipate exactly this), otherwise
   keep the longer span. Never double-mask the same bytes.
6. Per finding, classify: `policy.engine.classify_entity(ty)` ->
   - `Mask`: `policy.vault.intern(&Secret::new(value), ty, ns)` -> replace the span with the
     returned placeholder display; collect the `MappingRef`.
   - `IrreversibleRedact`: replace with a fixed typed marker (e.g. `[REDACTED:PASSWORD]`);
     **never call intern** -- the acceptance criterion "irreversible class never vault-stored"
     is tested, not assumed.
   - `Block` (entity-level): treat as IrreversibleRedact for the span (do not fail the whole
     artefact for one entity) and record it in stats.
   - `Pass`: leave the bytes untouched.
   Apply replacements back-to-front (descending span order) so earlier offsets stay valid.
7. Build the `MaskedPack { text, mapping_refs, stats, policy_version }` and ONE
   `AuditEvent::Scan` (counts by entity type, detector version, latency_us measured around the
   detect step); write it via `policy.audit` and also return it per the signature.
8. **Hard requirement (T05 review): demask logging has one owner.** The vault's own
   `demask_event` table already records every `resolve` attempt. When T07 wires the allowed
   path of `rehydrate` (`policy.vault.resolve` for non-hard-deny destinations after
   `policy.engine.demask_allowed` passes), do NOT also emit `AuditEvent::DemaskDecision` from
   the pipeline -- record in docs/decisions.md that the vault owns demask attribution in
   Phase 1, and leave a comment at the call site saying so.

Tests (integration, in `crates/vg-core/tests/`, with dev-dependencies on the real crates):
- End-to-end mask on a realistic mixed fixture (emails + IBAN + a secret + a date + paths)
  through real detectors/parsers/policy fixtures/a temp-keyed real vault/a temp audit sink:
  same value -> same placeholder; `MaskedPack.text` contains placeholders, not values.
- `vg_core::conformance::assert_masked_pack_excludes_raw_values` as a property test over the
  fixture's raw values (the invariant is tested, not type-enforced -- every MaskedPack field
  is pub).
- An `.env`-hinted input is Blocked: audit event written, no content in the pack, nothing
  interned.
- An IrreversibleRedact-classed value (e.g. a password) is redacted and provably absent from
  the vault (mapping_count stays 0 for it).
- An e2e criterion bench (`benches/`) for `mask()` on the reference buffer against the 25ms
  p95 budget, plus extending the existing plain-#[test] latency gate pattern
  (crates/vg-detectors/tests/latency_gate.rs is the precedent) to cover the full pipeline.

This is a one-shot headless dispatch: there is no follow-up channel. If any exact
naming/shape choice is ambiguous, use your best judgment, follow the frozen contracts and the
existing conventions in vg-core, and record the assumption in docs/decisions.md rather than
asking a clarifying question.
