## Task Description

Implement `vg_core::traits::Parser` in `crates/vg-parsers/src/` (currently an empty stub
crate -- see its `lib.rs` docstring). The trait: `fn can_parse(&self, artefact: &ArtefactHint)
-> bool` and `fn parse(&self, buf: &[u8]) -> ParseResult` where `ParseResult { spans:
Vec<Span>, artefact_kind: ArtefactKind }` (see `crates/vg-core/src/traits.rs` and
`types.rs` for `Span`, `ArtefactHint`, `ArtefactKind`).

Build one module per format under `crates/vg-parsers/src/`: `json.rs`, `yaml.rs` (add
`serde_yaml` or similar), `toml.rs`, `csv.rs`, `log.rs` (generic log-line parsing), `diff.rs`
(git unified diff), `env.rs` (`.env` files), plus tree-sitter support for one source
language of your choice (add the relevant `tree-sitter`/`tree-sitter-<lang>` crates) --
if the choice is ambiguous, pick something simple and common (e.g. Rust) rather than
asking.

The hard contract, not optional: **never panic on malformed input** -- return best-effort
spans instead. Test this with genuinely adversarial buffers (empty, truncated UTF-8,
unbalanced delimiters/brackets, a JSON file that's actually binary), not just one
happy-path fixture per format. Use `vg_core::conformance::assert_parser_never_panics`
for this.

Cross-crate integration requirement (interface-contracts.md, added 2026-07-16): at least
one integration test must feed this task's real `Span` output into
`vg_detectors::all_detectors()` (Task T03, already merged) on a realistic fixture, and
explicitly assert/record in `docs/decisions.md` whether the detectors' `spans` parameter
currently being a no-op (`_spans` in every detector's signature, confirmed 2026-07-16) is
an expected stage-appropriate gap or a real one -- don't let this go unnoticed until T07.

This is a one-shot headless dispatch: there is no follow-up channel. If any exact
naming/shape/language choice above is ambiguous, use your best judgment, follow existing
`vg-core`/`interface-contracts.md` conventions, and record the assumption in
`docs/decisions.md` rather than asking a clarifying question.
