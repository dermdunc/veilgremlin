## Task Description

Seeded corpus (labelled) + metrics harness in `vg-bench`; wire `vg_core::benchmark`
(currently `todo!`); Go/No-Go report generator; `vg bench` CLI subcommand.

CONTRACT CONSTRAINT: `benchmark(corpus: &Corpus, policy: &Policy) -> Metrics` and
`Metrics { recall, precision, false_positive_rate, p95_latency_us }` are frozen (v1.3 of
docs/architecture/interface-contracts.md — read it FIRST; note §8 is the v1.3 hook
protocol, exit 0 + JSON transforms / exit 2 block). If the harness genuinely needs a
richer shape, use the contract-change protocol (agent-factory-plan.md §6) with rationale
— do not smuggle extra channels through side effects.

BANKED MEASUREMENTS this task owns (deferred debts — each needs a labelled corpus slice
and a number in the report, not prose):
1. Entropy- and phone-detector false-positive rates (deferred since Wave B: the
   detectors landed with thresholds chosen by eye, never measured).
2. Zero-raw-PII property: run `vg_core::conformance::assert_masked_pack_excludes_raw_values`
   over every corpus sample as a harness assertion (the §1 invariant's promised
   property test).
3. Display-collision incidence (T09 doubt round): corpus MUST include raw text that
   literally contains placeholder-shaped strings (`EMAIL_001`, spoofed `EMAIL_999`) both
   with and without the pack minting the same display; measure demask corruption. If
   incidence > 0 on realistic slices, propose collision-avoiding minting at intern time
   (skip an ordinal whose display occurs in the raw buffer) as a T11 decision with data.
4. Env-shaped content WITHOUT a file_path hint (T09 known limit: Bash `echo X > .env`
   bypasses the artefact Block): measure what entity-level detection alone catches on
   dotenv-shaped corpus lines so the residual risk is quantified.
5. Hook end-to-end latency: p95 of a COLD `vg hook` invocation (process spawn +
   SQLCipher open + policy load + mask) measured at the binary level, not just the
   in-process mask path — the runbook claims "tens of ms"; verify or correct it. The
   frozen Metrics carries one p95_latency_us: use it for the full hook path and report
   the in-process split in the report text.
6. Dead-config detection: the default policy's `by_language: dotenv` branch is
   unreachable (`hint.language_id` is never set anywhere) — either wire language_id from
   the parsers where structurally sound, or have the report flag dead policy branches
   explicitly. Do not silently leave policy that cannot fire.

CORPUS REQUIREMENTS: labelled (expected findings per sample); include the two documented
YAML under-spanning reproducers (see docs/decisions.md T08 entry), dotfiles, JSON tool
payloads whose masking must survive the v1.3 `same_shape` guard (string values only),
multi-entity lines with overlapping spans, and benign look-alike slices (EMAIL_0015-style
near-collisions). Fixture values follow the synthetic-data rule: RFC-2606 domains,
published test IBANs, fake-shaped tokens — never real-shaped-and-registrable data.

HARD REQUIREMENTS (banked from earlier reviews, still binding): detectors scan the full
raw buffer (spans enrichment-only); demask resolves exclusively via pack bindings;
harness/report output must never print a raw detected value (report refs/counts only —
the eval tool must satisfy the same redaction discipline it measures).
